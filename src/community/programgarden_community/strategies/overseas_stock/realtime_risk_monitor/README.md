# 실시간 위험 관리 전략

## 전략 ID
`realtime_risk_monitor`

## 버전
1.0.0

## 기여자
ProgramGarden Team

## 간단한 설명
WebSocket을 통해 실시간 틱 데이터를 수신하며 수익률을 계산하고, 위험 한도 초과 시 자동으로 청산합니다.

## 이 전략이 필요한 이유

### 문제점
- 실시간 시장 변동에 즉각 대응하지 못함
- 손실이 커지는 것을 인지하지 못하고 방치
- 감정적으로 손절 결정을 미루다가 큰 손실
- 과매매로 인한 수수료 및 슬리피지 손실

### 해결책
- 실시간 P&L 모니터링으로 즉각 대응
- 객관적 위험 규칙 기반 자동 청산
- 일일 거래 횟수 제한으로 과매매 방지
- 연속 손실 감지로 틸팅 방지

## 위험 관리 규칙

| 규칙 | 기본값 | 동작 |
|------|--------|------|
| 일일 손실 한도 | -5% | 모든 포지션 청산 + 24시간 거래 중단 |
| 포지션 손실 한도 | -3% | 해당 포지션 청산 |
| 일일 거래 횟수 | 20회 | 당일 추가 거래 차단 |
| 연속 손실 | 3연패 | 당일 거래 중단 |

## 워크플로우

```
┌─────────────────────────────────────────────────────────────┐
│  WebSocket 연결                                             │
│  BrokerNode (websocket mode) → ScheduleNode (realtime)     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  실시간 데이터 수신                                          │
│  RealMarketDataNode → tick, quote, trade                    │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  P&L 계산                                                   │
│  PnLCalculatorNode                                          │
│  • daily_pnl (실현 + 미실현)                                │
│  • position_pnl (개별 포지션)                               │
│  • trade_count (일일 거래)                                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  위험 규칙 체크 (OR 로직)                                   │
│  ├── DailyLossCheck (-5%)                                  │
│  ├── PositionLossCheck (-3%)                               │
│  ├── DailyTradeCheck (20회)                                │
│  └── ConsecutiveLossCheck (3연패)                          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  하나라도 충족 시 → 긴급 청산 + 거래 중단 + 알림            │
└─────────────────────────────────────────────────────────────┘
```

## 파라미터

| 이름 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `symbols` | list | ["SOXS"] | 모니터링 대상 종목 |
| `daily_loss_limit` | float | -5 | 일일 손실 한도 (%) |
| `position_loss_limit` | float | -3 | 포지션 손실 한도 (%) |
| `max_daily_trades` | int | 20 | 일일 최대 거래 횟수 |
| `max_consecutive_losses` | int | 3 | 연속 손실 제한 |
| `risk_check_interval` | int | 1 | 위험 체크 간격 (초) |

## 사용법

```python
from programgarden_community.strategies import get_strategy
from programgarden import ProgramGarden

strategy = get_strategy("overseas_stock", "realtime_risk_monitor")

# 커스텀 위험 한도 설정
strategy["inputs"]["daily_loss_limit"]["default"] = -3  # 더 보수적인 -3%

pg = ProgramGarden()
job = pg.run(strategy)
```

## 주의사항

- 실시간 데이터는 네트워크 지연이 있을 수 있습니다
- 급격한 시장 변동 시 청산 가격이 예상과 다를 수 있습니다
- 거래 중단은 수동으로 해제할 수 있습니다
- 위험 한도는 본인의 자금 규모에 맞게 조정하세요
