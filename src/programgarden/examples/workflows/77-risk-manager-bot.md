# 77 - 통합 리스크 관리 봇: 손절 + 익절 + 포트폴리오 경고 (선물 모의)

> **시나리오**: "포지션별 -5% 손절 + +10% 익절 + 계좌 전체 낙폭 감시까지 한꺼번에 자동화해줘."

## 아키텍처: 3중 병렬 게이트

```
                      ┌─► IfNode(pnl ≤ -5%) ──► Sell → Telegram (손절)
                      │
Schedule → Account ───┼─► IfNode(pnl ≥ +10%) ─► Sell → Telegram (익절)
                      │
                      └─► Portfolio → IfNode(drawdown ≥ 10%) → Telegram (경고)
```

하나의 AccountNode 에서 **3개의 독립 분기**로 리스크 요소를 병렬 감시. 각 분기는 서로 간섭 없이 동시에 작동합니다.

## 3중 안전장치

### 1. 손절 (-5%)
- **목표**: 큰 손실 차단
- **트리거**: 포지션별 `pnl_rate ≤ -5%`
- **액션**: 해당 포지션 전량 시장가 매도

### 2. 익절 (+10%)
- **목표**: 수익 확정
- **트리거**: 포지션별 `pnl_rate ≥ +10%`
- **액션**: 해당 포지션 전량 시장가 매도

### 3. 포트폴리오 낙폭 경고 (≥10%)
- **목표**: 계좌 전체 위험 감지
- **트리거**: `PortfolioNode.drawdown_percent ≥ 10%`
- **액션**: 텔레그램 경고 (주문 없음 — 수동 개입 유도)

## Per-position Auto-iterate vs 집계 평가

| 게이트 | 입력 | 평가 방식 |
|--------|------|-----------|
| 손절 | `Account.positions` (리스트) | 포지션별 개별 평가 (auto-iterate) |
| 익절 | `Account.positions` (리스트) | 포지션별 개별 평가 (auto-iterate) |
| 포트폴리오 | `PortfolioNode.drawdown_percent` (스칼라) | 계좌 전체 1회 평가 |

## PortfolioNode 의 역할

```json
{
  "id": "portfolio_risk",
  "type": "PortfolioNode",
  "max_drawdown_percent": 15
}
```

- `max_drawdown_percent: 15` — 15% 도달 시 자동 **킬스위치** (모든 매매 중단)
- `drawdown_percent` 출력 — 현재 낙폭 (HWM 대비)
- 내부적으로 `risk_tracker` 에 HWM 자동 기록 (`_risk_features: {"hwm"}`)

## 스케줄 주기

`*/3 * * * *` — 3분마다 체크. 더 빠른 반응이 필요하면:

| cron | 간격 |
|------|------|
| `*/1 * * * *` | 1분 (고주파) |
| `*/3 * * * *` | 3분 (기본) |
| `*/5 * * * *` | 5분 (균형) |
| `*/15 * * * *` | 15분 (저주파) |

## 실전 활용 시나리오

다른 워크플로우가 매수만 담당하고, 이 워크플로우가 **백그라운드 리스크 매니저** 로 계속 돌면서 자동 청산/경고를 수행합니다:

```
[진입 봇] (ex. #73, #76) → 매수만 담당
[77 리스크 매니저]        → 청산 + 경고 담당 (상시 가동)
```

## 확장 아이디어

- **단계별 손절**: IfNode 3단 (-3/-5/-10) 각각 25%/50%/100% 매도
- **트레일링 스탑**: `trailing_stop` 플러그인 추가 → HWM 대비 드로다운
- **포지션 크기 제한**: `max_position_limit` 플러그인으로 신규 진입 제한
- **켈리 사이징**: `kelly_criterion` 플러그인으로 다음 진입 수량 계산

## 주의사항

- **손절/익절 동시 트리거 불가** — `pnl_rate` 가 -5% 이하 AND +10% 이상 불가
- 텔레그램 발송 후 즉시 주문하지 않고 IfNode 조건 충족 포지션만 주문 → 정상
- 보유 포지션 없으면 모든 IfNode 가 false → 조용히 종료 (정상)
- `paper_trading: true` — 모의 환경. 실전은 실계좌 credential 로 전환
