# 75 - 데이트레이딩 자동화: 장시작 매수 + 장마감 청산 (선물 모의)

> **시나리오**: "매일 장 시작하면 알아서 사고, 장 끝나기 전에 알아서 팔아서 오버나이트 리스크 없애줘."

## 아키텍처: 2중 스케줄 병렬

```
                     ┌── schedule_entry(09:30) → Watchlist → Historical → TSMOM → Buy → Telegram
Start → Broker ──────┤
                     └── schedule_exit(15:55) → Account → Sell(전량) → Telegram
```

하나의 BrokerNode 에서 두 개의 독립된 ScheduleNode 로 분기 → **병렬 실행**.

## 진입 로직 (09:30 HKT)

1. 감시 종목 1개 (`HMHJ26`)
2. 최근 20일 일봉 조회
3. **TimeSeriesMomentum(10일)** — 10일 수익률 양수면 binary signal=1
4. 신호 양수 시 시장가 1계약 매수
5. 텔레그램 알림

## 청산 로직 (15:55 HKT)

1. 보유 포지션 전부 조회
2. 각 포지션 `close_side` 로 반대매매 주문 (auto-iterate)
3. 텔레그램으로 종목별 청산 결과 통지

## cron 표현식 해석

| 값 | 의미 |
|----|------|
| `30 9 * * 1-5` | 평일 09:30 |
| `55 15 * * 1-5` | 평일 15:55 |
| timezone: `Asia/Hong_Kong` | HKEX 현지시각 기준 |

## 왜 오버나이트 리스크를 피해야 하나?

- **갭 리스크**: 장외 시간에 악재 발생 시 다음날 시가가 급락/급등 할 수 있음
- **마진콜 방지**: 선물은 평가손실이 증거금을 초과하면 강제청산
- **심리적 안정**: 포지션 없이 밤 보내기 → 편안한 수면

## 확장 아이디어

- **여러 종목**: entry_watchlist 에 종목 추가 → 각 종목별 독립 진입
- **부분 청산**: 장 마감 대신 일부만 청산, 일부는 홀드
- **진입 지연**: 09:30 대신 10:00 → 장 시작 변동성 회피
- **TradingHoursFilter 추가**: 휴장일 자동 스킵

## 주의사항

- **max_duration_hours: 24** — 24시간 후 스케줄 자동 종료 (프로세스 재시작 필요)
- 실전 데이트레이딩은 수수료/슬리피지로 수익률 악화 — 모의로 충분히 테스트
- HKEX 미니선물 정규장: 09:15-12:00 (AM) + 13:00-16:30 (PM) — 장 마감은 16:30, 여기선 15:55 로 여유 확보
