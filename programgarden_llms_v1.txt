## 1. 프로젝트 개요
================================================================================

ProgramGarden은 코딩 지식이 없는 시스템 트레이더와 투자자들이 손쉽게 사용할 수 있는
**개인 맞춤형 자동화매매 오픈소스 플랫폼**입니다.

### 핵심 특징
- DSL(Domain Specific Language)을 통한 선언적 자동매매 전략 정의
- 해외주식(overseas_stock) 및 해외선물(overseas_futures) 지원
- LS증권 OpenAPI 연동
- 커뮤니티 기반 전략 공유 생태계
- AI 챗봇을 통한 전략 코드 생성 지원

### PyPI 패키지
- programgarden: https://pypi.org/project/programgarden/
- programgarden-core: https://pypi.org/project/programgarden-core/
- programgarden-finance: https://pypi.org/project/programgarden-finance/
- programgarden-community: https://pypi.org/project/programgarden-community/

### 설치
```bash
pip install programgarden
# 또는
poetry add programgarden
```

### 요구사항
- Python 3.10 이상
- LS증권 API 키 (appkey, appsecretkey)

## 3. DSL 시스템 구조 (★ 가장 중요)
================================================================================

ProgramGarden DSL은 4개의 최상위 섹션으로 구성됩니다:

```python
{
    "settings": { ... },      # 시스템 메타정보
    "securities": { ... },    # 증권사 연결 정보
    "strategies": [ ... ],    # 종목 분석 전략 (선택)
    "orders": [ ... ]         # 주문 전략 (선택)
}
```

### 3.1 settings 섹션 (필수)

```python
{
    "settings": {
        "system_id": "my_system_001",        # 필수: 고유 시스템 ID
        "name": "추세 추종 전략",              # 전략 이름
        "description": "이동평균 기반 매매",   # 설명
        "version": "1.0.0",                  # 버전
        "author": "홍길동",                   # 작성자
        "date": "2025-12-15",                # 작성일
        "debug": "DEBUG",                    # 로그 레벨: TRACE|DEBUG|INFO|WARN|ERROR
        
        # 드라이런 모드 (선택)
        "dry_run_mode": "test",              # test|live|guarded_live
        
        # 성능 임계치 (guarded_live 모드용)
        "perf_thresholds": {
            "max_avg_cpu_percent": 80,
            "max_memory_delta_mb": 256
        }
    }
}
```

#### dry_run_mode 설명
- `test`: 실제 주문 없이 전략 테스트만 수행. 성공 시 자동으로 live로 승격
- `live`: 실제 주문 즉시 전송 (기본값)
- `guarded_live`: 실주문 전송하되, 성능 임계치 초과 시 즉시 중단

### 3.2 securities 섹션 (필수)

```python
{
    "securities": {
        "company": "ls",                     # 필수: 증권사 (현재 ls만 지원)
        "product": "overseas_stock",         # 필수: overseas_stock 또는 overseas_futures
        "appkey": "YOUR_APPKEY",             # 필수: LS증권 앱키
        "appsecretkey": "YOUR_APPSECRET",    # 필수: LS증권 시크릿키
        "paper_trading": False               # 선택: 모의투자 여부 (기본: False)
    }
}
```

#### product 값
- `overseas_stock`: 해외주식 (나스닥, NYSE 등)
- `overseas_futures`: 해외선물 (CME, HKEX 등)

⚠️ 주의: 해외선물 모의투자는 HKEX(홍콩거래소)만 지원됩니다.

### 3.3 strategies 섹션 (종목 분석 전략)

```python
{
    "strategies": [
        {
            "id": "trend_strategy_001",           # 필수: 전략 ID (고유)
            "description": "골든크로스 전략",       # 설명
            
            # 스케줄 설정
            "schedule": "0 */15 * * * *",         # 크론 표현식 (6필드: 초 분 시 일 월 요일)
            "timezone": "Asia/Seoul",             # 시간대
            "run_once_on_start": True,            # 시작 시 즉시 1회 실행
            
            # 논리 연산자 설정
            "logic": "at_least",                  # all|any|not|xor|at_least|at_most|exactly|weighted
            "threshold": 1,                       # at_least/at_most/exactly/weighted에서 사용
            
            # 대상 종목
            "symbols": [
                {"symbol": "TSLA", "exchange": "NASDAQ"},
                {"symbol": "NVDA", "exchange": "NASDAQ"}
            ],
            
            # 종목 수 제한 (선택)
            "max_symbols": {
                "order": "mcap",                  # mcap(시가총액순) 또는 random
                "limit": 5                        # 최대 종목 수
            },
            
            # 주문 전략 연결
            "order_id": "buy_order_001",          # orders 섹션의 order_id 참조
            
            # 조건 목록
            "conditions": [
                {
                    "condition_id": "SMAGoldenDeadCross",  # 커뮤니티 전략 ID
                    "weight": 0.6,                         # weighted 로직용 가중치 (선택)
                    "params": {                            # 전략별 파라미터
                        "short_period": 5,
                        "long_period": 20,
                        "alignment": "golden"
                    }
                }
            ]
        }
    ]
}
```

#### logic 연산자 상세
| 연산자 | 설명 | threshold 필요 |
|--------|------|----------------|
| all | 모든 조건 만족 (AND) | X |
| any | 하나 이상 만족 (OR) | X |
| not | 모든 조건 불만족 | X |
| xor | 정확히 하나만 만족 | X |
| at_least | N개 이상 만족 | O |
| at_most | N개 이하 만족 | O |
| exactly | 정확히 N개 만족 | O |
| weighted | 가중치 합이 threshold 이상 | O |

### 3.4 orders 섹션 (주문 전략)

```python
{
    "orders": [
        {
            "order_id": "buy_order_001",          # 필수: 주문 전략 ID (고유)
            "description": "분할 매수 전략",       # 설명
            "block_duplicate_buy": True,          # 중복 매수 방지
            
            # 주문 시간대 제한 (선택)
            "order_time": {
                "start": "22:00:00",              # 시작 시간
                "end": "05:30:00",                # 종료 시간
                "days": ["mon", "tue", "wed", "thu", "fri"],  # 요일
                "timezone": "Asia/Seoul",
                "behavior": "defer",              # defer(대기) 또는 skip(건너뛰기)
                "max_delay_seconds": 3600         # 최대 대기 시간(초)
            },
            
            # 주문 조건 (커뮤니티 전략 또는 커스텀 클래스)
            "condition": {
                "condition_id": "StockSplitFunds",    # 커뮤니티 전략 ID
                "params": {
                    "percent_balance": 10.0,          # 예수금 대비 비율
                    "max_symbols": 5                  # 최대 종목 수
                }
            }
        }
    ]
}
```

## 5. 실행 방법
================================================================================

### 5.1 기본 실행

```python
from programgarden import Programgarden
import os

# 인스턴스 생성
pg = Programgarden()

# 콜백 설정
pg.on_strategies_message(
    callback=lambda msg: print(f"전략 결과: {msg}")
)
pg.on_real_order_message(
    callback=lambda msg: print(f"주문 결과: {msg}")
)
pg.on_error_message(
    callback=lambda msg: print(f"에러: {msg}")
)
pg.on_performance_message(
    callback=lambda msg: print(f"성능: {msg}")
)

# 실행
pg.run(system={
    "settings": {
        "system_id": "example_001",
        "name": "예제 전략",
        "debug": "DEBUG"
    },
    "securities": {
        "company": "ls",
        "product": "overseas_stock",
        "appkey": os.getenv("APPKEY"),
        "appsecretkey": os.getenv("APPSECRET")
    },
    "strategies": [...],
    "orders": [...]
})
```

### 5.2 비동기 종료

```python
import asyncio

# 종료
await pg.stop()
```

### 5.3 환경 변수 설정 (.env 파일)

```bash
# 해외주식용
APPKEY=your_stock_appkey
APPSECRET=your_stock_appsecret

# 해외선물용 (별도 키 필요)
APPKEY_FUTURE=your_futures_appkey
APPSECRET_FUTURE=your_futures_appsecret
```

## 6. 스케줄 표현식 (Cron)
================================================================================

### 6필드 형식: 초 분 시 일 월 요일

| 필드 | 허용값 |
|------|--------|
| 초 | 0-59 |
| 분 | 0-59 |
| 시 | 0-23 |
| 일 | 1-31, l(마지막) |
| 월 | 1-12, jan-dec |
| 요일 | 0-6 (0=일요일), sun-sat |

### 연산자
- `*`: 모든 값
- `A,B`: 나열 (예: mon,wed,fri)
- `A-B`: 범위 (예: 9-17)
- `*/N`: N간격 (예: */5 = 5마다)
- `l`: 마지막 (일 필드)
- `요일#N`: N번째 요일 (예: 2#3 = 셋째 화요일)

### 자주 쓰는 예시
```
* * * * * *        # 매초
*/5 * * * * *      # 5초마다
0 * * * * *        # 매분 0초
0 */15 * * * *     # 15분마다
0 0 * * * *        # 매시 정각
0 30 9 * * *       # 매일 09:30:00
0 0 9 * * mon-fri  # 평일 09:00:00
0 0 22 * * *       # 매일 22:00:00 (미국 장 시작)
```

## 8. 커스텀 전략 개발 (★ 개발자용)
================================================================================

### 8.1 종목추출 조건 (Strategy Condition) 개발

#### 해외주식 조건 클래스
```python
from programgarden_core import (
    BaseStrategyConditionOverseasStock,
    BaseStrategyConditionResponseOverseasStockType,
)

class MyStockCondition(BaseStrategyConditionOverseasStock):
    # 필수 클래스 속성
    id = "MyStockCondition"
    description = "내 커스텀 주식 조건"
    securities = ["ls-sec.co.kr"]
    
    def __init__(self, my_param: int = 10):
        super().__init__()  # 반드시 호출!
        self.my_param = my_param
    
    async def execute(self) -> BaseStrategyConditionResponseOverseasStockType:
        # self.symbol에 현재 평가 중인 종목 정보가 주입됨
        symbol = self.symbol or {}
        
        # 조건 검증 로직 작성
        is_pass = True  # 실제 로직으로 대체
        
        # 반드시 이 형식으로 반환
        return {
            "condition_id": self.id,
            "success": is_pass,                    # 조건 통과 여부
            "symbol": symbol.get("symbol", ""),    # 종목 코드
            "exchcd": symbol.get("exchcd", ""),    # 거래소 코드
            "data": {"my_data": self.my_param},    # 추가 데이터
            "weight": 1 if is_pass else 0,         # 가중치 (0~1)
            "product": "overseas_stock",           # 상품 유형
        }
```

#### 해외선물 조건 클래스
```python
from programgarden_core import (
    BaseStrategyConditionOverseasFutures,
    BaseStrategyConditionResponseOverseasFuturesType,
)

class MyFuturesCondition(BaseStrategyConditionOverseasFutures):
    id = "MyFuturesCondition"
    description = "내 커스텀 선물 조건"
    securities = ["ls-sec.co.kr"]
    
    def __init__(self, threshold: float = 0.5):
        super().__init__()
        self.threshold = threshold
    
    async def execute(self) -> BaseStrategyConditionResponseOverseasFuturesType:
        symbol = self.symbol or {}
        
        # 선물은 position_side 필수!
        score = 0.8  # 실제 로직으로 대체
        
        if score >= self.threshold:
            position_side = "long"
        elif score <= -self.threshold:
            position_side = "short"
        else:
            position_side = "flat"  # 또는 "neutral"
        
        return {
            "condition_id": self.id,
            "success": position_side in {"long", "short"},
            "symbol": symbol.get("symbol", ""),
            "exchcd": symbol.get("exchcd", ""),
            "data": {"score": score},
            "weight": abs(score),
            "product": "overseas_futures",
            "position_side": position_side,  # 선물 필수!
        }
```

#### position_side 값 설명 (선물 전용)
| 값 | 의미 |
|----|------|
| long | 매수 방향 진입 |
| short | 매도 방향 진입 |
| flat | 진입 신호 없음 (조건 실패 처리) |
| neutral | 방향 중립, 다른 조건에 위임 (필터용) |

⚠️ 모든 조건이 neutral이면 방향을 결정할 수 없어 주문이 실행되지 않습니다.

#### 해외주식 신규주문
```python
from typing import List
from programgarden_core import (
    BaseNewOrderOverseasStock,
    BaseNewOrderOverseasStockResponseType,
)

class MyNewOrder(BaseNewOrderOverseasStock):
    id = "MyNewOrder"
    description = "내 신규주문 전략"
    securities = ["ls-sec.co.kr"]
    order_types = ["new_buy", "new_sell"]  # 지원하는 주문 타입
    
    def __init__(self, percent: float = 10.0):
        super().__init__()
        self.percent = percent
    
    async def execute(self) -> List[BaseNewOrderOverseasStockResponseType]:
        # 자동 주입되는 속성들:
        # self.available_symbols: 주문 가능한 종목 리스트
        # self.held_symbols: 보유 종목 리스트
        # self.non_traded_symbols: 미체결 주문 리스트
        # self.dps: 예수금 정보
        # self.system_id: 시스템 ID
        
        if not self.available_symbols:
            return []
        
        orders = []
        for symbol in self.available_symbols:
            orders.append({
                "success": True,
                "ord_ptn_code": "02",      # 01=매도, 02=매수
                "ord_mkt_code": symbol.get("exchcd", "82"),  # 81=NYSE, 82=NASDAQ
                "shtn_isu_no": symbol.get("symbol", ""),
                "ord_qty": 1,              # 주문 수량
                "ovrs_ord_prc": 100.0,     # 주문 가격
                "ordprc_ptn_code": "00",   # 00=지정가, 03=시장가
                "brk_tp_code": "",
                "crcy_code": "USD",
                "pnl_rat": 0.0,
                "pchs_amt": 0.0,
                "bns_tp_code": "2",        # 1=매도, 2=매수
            })
        return orders
    
    async def on_real_order_receive(self, order_type, response):
        # 실시간 주문 이벤트 처리
        pass
```

```python
class MomentumCondition(BaseStrategyConditionOverseasFutures):
    id = "MomentumCondition"
    description = "모멘텀 기반 롱/숏 판단"
    securities = ["ls-sec.co.kr"]
    
    def __init__(self, threshold: float = 0.5):
        super().__init__()
        self.threshold = threshold
    
    async def execute(self) -> BaseStrategyConditionResponseOverseasFuturesType:
        symbol = self.symbol or {}

        momentum = 0.7
        
        if momentum >= self.threshold:
            position_side = "long"
        elif momentum <= -self.threshold:
            position_side = "short"
        else:
            position_side = "flat"
        
        return {
            "condition_id": self.id,
            "success": position_side in {"long", "short"},
            "symbol": symbol.get("symbol", ""),
            "exchcd": symbol.get("exchcd", "HKEX"),
            "data": {"momentum": momentum},
            "weight": abs(momentum),
```

## 14. 주의사항 및 FAQ
================================================================================

### ⚠️ 중요 주의사항

1. **테스트 버전 경고**
   - ProgramGarden은 현재 테스트 버전입니다
   - 정식 버전 출시 전까지 모의투자 또는 소액 투자를 권장합니다
   - 오픈소스 사용 시 발생하는 문제에 대한 책임은 사용자에게 있습니다

2. **API 키 보안**
   - appkey, appsecretkey를 코드에 직접 작성하지 마세요
   - .env 파일이나 환경 변수를 사용하세요
   - .gitignore에 .env를 추가하세요

3. **모의투자 제한**
   - 해외선물 모의투자는 HKEX(홍콩거래소)만 지원됩니다
   - CME, EUREX 등은 실거래만 가능합니다

4. **TypedDict 반환 형식 준수**
   - 커스텀 클래스 개발 시 반드시 지정된 TypedDict 형식을 따라야 합니다
   - 필수 키가 누락되면 런타임 에러가 발생합니다

5. **선물 position_side 필수**
   - 해외선물 조건에서는 반드시 position_side를 반환해야 합니다
   - 모든 조건이 neutral이면 방향을 알 수 없어 주문이 실행되지 않습니다

### FAQ

**Q: 지원되는 증권사는?**
A: 현재 LS증권만 지원됩니다.

**Q: 국내 주식은 지원하나요?**
A: 아직 지원되지 않습니다. 추후 업데이트 예정입니다.

**Q: 코인 거래는 지원하나요?**
A: 아직 지원되지 않습니다. 추후 업데이트 예정입니다.

**Q: 백테스팅은 가능한가요?**
A: 현재 버전에서는 제공되지 않습니다.

**Q: 여러 전략을 동시에 실행할 수 있나요?**
A: 네, strategies 배열에 여러 전략을 정의하면 됩니다.

**Q: 조건 중첩이 가능한가요?**
A: 네, conditions 내부에 다시 conditions를 정의할 수 있습니다.

ProgramGarden Finance는 LS증권 OpenAPI를 Python 친화적으로 감싼 라이브러리입니다.

### 17.1 주요 특징

- **간편한 LS증권 API 통합**: 복잡한 OpenAPI 스펙을 상황별 클래스로 추상화

- **비동기·동기 동시 지원**: 모든 API에 동기(`req`)와 비동기(`req_async`) 호출 제공
- **토큰 자동 관리**: OAuth 토큰 발급 및 만료 시 자동 갱신
- **타입 안전성**: Pydantic 기반 요청/응답 모델로 타입 힌트 강화

### 17.6 실시간 시세(WebSocket) 구독

```python
import asyncio
import os
from dotenv import load_dotenv
from programgarden_finance import LS
import logging

load_dotenv()

async def subscribe_realtime():
    ls = LS()
    if not ls.login(appkey=os.getenv("APPKEY"), appsecretkey=os.getenv("APPSECRET")):
        logging.error("로그인 실패")
        return

    def on_message(resp):
        print(f"실시간 데이터: {resp}")

    client = ls.overseas_stock().real()
    await client.connect()

    # GSC: 실시간 체결
    gsc = client.GSC()
    gsc.add_gsc_symbols(symbols=["81SOXL", "82TSLA"])
    gsc.on_gsc_message(on_message)

    # 구독 유지
    await asyncio.sleep(3600)
    
    # 구독 해지
    gsc.on_remove_gsc_message()
    await client.close()

asyncio.run(subscribe_realtime())
```

### 19.1 기본 예외 클래스

```python
from programgarden_core.exceptions import BasicException

class BasicException(Exception):
    """모든 ProgramGarden 예외의 기본 클래스"""
    def __init__(
        self,
        message: str = "알 수 없는 오류가 발생했습니다.",
        code: str = "UNKNOWN_ERROR",
        data: Optional[Dict[str, Any]] = None,
    ):
        self.code = code
        self.message = message
        self.data = data or {}
    
    def to_payload(self, extra=None) -> Dict[str, Any]:
        """예외를 딕셔너리로 변환"""
        return {
            "code": self.code,
            "message": self.message,
            "data": {**self.data, **(extra or {})}
        }
```

### 20.3 전략 클래스 템플릿

```python
from pydantic import BaseModel, Field
from programgarden_core import (
    BaseStrategyConditionOverseasStock,
    BaseStrategyConditionResponseOverseasStockType,
)

class MySMAConditionParams(BaseModel):
    """파라미터 정의"""
    short_period: int = Field(5, gt=0, title="단기 이평 기간")
    long_period: int = Field(20, gt=0, title="장기 이평 기간")

class MySMACondition(BaseStrategyConditionOverseasStock):
    # 필수 클래스 속성
    id: str = "MySMACondition"
    name: str = "나만의 SMA 기반 컨디션"
    description: str = "SMA 교차를 감지하여 매수/매도 신호를 생성합니다."
    securities: list = ["ls-sec.co.kr"]
    parameter_schema: dict = MySMAConditionParams.model_json_schema()

    def __init__(self, short_period: int = 5, long_period: int = 20, **kwargs):
        super().__init__()
        self.short_period = short_period
        self.long_period = long_period

    async def execute(self) -> BaseStrategyConditionResponseOverseasStockType:
        symbol = self.symbol or {}
        
        # 실제 SMA 계산 로직 작성
        is_golden_cross = True  # 예시
        
        return {
            "condition_id": self.id,
            "success": is_golden_cross,
            "symbol": symbol.get("symbol", ""),
            "exchcd": symbol.get("exchcd", ""),
            "data": {
                "short_period": self.short_period,
                "long_period": self.long_period
            },
            "weight": 1 if is_golden_cross else 0,
            "product": "overseas_stock",
        }

# 필수: __all__ 정의
__all__ = ["MySMACondition"]
```

### 23.1 표준 logging 모듈 사용

ProgramGarden은 Python 표준 `logging` 모듈을 사용합니다.
기본값은 NullHandler로 설정되어 있어 로그가 출력되지 않으며,
필요한 개발자가 직접 핸들러를 추가하여 활성화할 수 있습니다.

```python
import logging

# 로깅 활성화 (기본 설정)
logging.basicConfig(level=logging.DEBUG)

# 또는 ProgramGarden 네임스페이스만 활성화
logging.getLogger('pg').setLevel(logging.DEBUG)
logging.getLogger('pg').addHandler(logging.StreamHandler())
```

```python
import logging
```

## 24. 조건 중첩 (Nested Conditions)
================================================================================

### 24.1 기본 중첩 구조

조건 내부에 또 다른 조건을 정의할 수 있습니다:

```python
{
    "strategies": [
        {
            "id": "nested_example",
            "logic": "all",
            "conditions": [
                {
                    "condition_id": "SMAGoldenDeadCross",
                    "params": {"short_period": 5, "long_period": 20},
                    
                    # 중첩 조건 (하위 조건이 모두 통과해야 상위 조건 평가)
                    "conditions": [
                        {
                            "condition_id": "VolatilityFilter",
                            "params": {"min_atr": 0.5},
                            
                            # 더 깊은 중첩도 가능
                            "conditions": [
                                {
                                    "condition_id": "VolumeSpike",
                                    "params": {"threshold": 2.0}
                                }
                            ]
                        }
                    ]
                }
            ]
        }
    ]
}
```

### 24.2 중첩 실행 순서

1. 가장 안쪽(깊은) 조건부터 평가
2. 하위 조건이 모두 통과해야 상위 조건 평가
3. 하위 조건 실패 시 상위 조건은 평가하지 않음 (short-circuit)

## 26. 커뮤니티 전략 레퍼런스

### 26.1 커뮤니티 전략 안내

커뮤니티 전략은 외부 기여자들에 의해 지속적으로 추가되고 업데이트됩니다.
따라서 이 문서에서 전략 목록을 고정적으로 나열하지 않습니다.

**최신 커뮤니티 전략 목록은 아래 GitHub 링크를 참조하세요:**

📎 https://github.com/programgarden/programgarden/tree/main/docs/strategies

위 링크에서 다음 정보를 확인할 수 있습니다:
- 사용 가능한 모든 전략 목록
- 각 전략의 파라미터 정의
- 사용 예시 코드
- 전략별 신호 해석 방법

### 26.2 전략 사용 기본 형식

모든 커뮤니티 전략은 동일한 DSL 형식으로 사용됩니다:

```python
{
    "condition_id": "전략ID",
    "params": {
        "파라미터1": 값1,
        "파라미터2": 값2
    }
}
```

### 26.3 전략 종류

커뮤니티 전략은 크게 4가지 유형으로 분류됩니다:

| 유형 | 설명 | 용도 |
|------|------|------|
| strategy_conditions | 종목추출 조건 | 매매 대상 종목 선정 |
| new_order_conditions | 신규주문 조건 | 매수/매도 주문 생성 |
| modify_order_conditions | 정정주문 조건 | 기존 주문 수정 |
| cancel_order_conditions | 취소주문 조건 | 미체결 주문 취소 |

### 26.4 전략 파라미터 조회 방법

각 전략의 파라미터는 Pydantic 모델로 정의되어 있어
JSON Schema 형태로 조회할 수 있습니다:

```python
from programgarden_community.overseas_stock.strategy_conditions import SMAGoldenDeadCross

# 파라미터 스키마 조회
schema = SMAGoldenDeadCross.parameter_schema
print(schema)
```

새로운 전략을 기여하고 싶다면:
1. 기여 가이드 문서 참조: https://programgarden.gitbook.io/docs/contribution
2. GitHub 저장소 Fork
3. 전략 코드 및 README.md 작성

### 27.7 터틀 트레이딩 시스템 전체 예제

```python
import asyncio
from dotenv import load_dotenv
import os
from programgarden import ProgramGarden, PGListener

load_dotenv()

# 이벤트 리스너
listener = PGListener()

@listener.on_system_ready
def on_ready(symbols):
    print(f"시스템 준비 완료. 종목: {len(symbols)}개")

@listener.on_strategy_executed
def on_strategy(strategy_id, results):
    print(f"전략 실행: {strategy_id}")
    for r in results:
        if r["success"]:
            print(f"  ✓ {r['symbol']} 조건 충족")

@listener.on_order_submitted
def on_order(order_info):
    print(f"주문 제출: {order_info['symbol']} {order_info['side']} {order_info['qty']}")

@listener.on_order_filled
def on_filled(fill_info):
    print(f"체결 완료: {fill_info['symbol']} @ {fill_info['price']}")

@listener.on_error
def on_error(error):
    print(f"에러 발생: {error}")

# 시스템 설정
system = {
    "settings": {
        "system_id": "turtle_system",
        "debug": "INFO",
        "product": "overseas_stock"
    },
    "securities": {
        "appkey": os.getenv("APPKEY"),
        "appsecretkey": os.getenv("APPSECRET"),
        "account_number": os.getenv("ACCOUNT_NUMBER"),
        "account_product_code": "01",
        "symbols": ["82TSLA", "82AAPL", "82NVDA", "82MSFT", "82GOOGL"]
    },
    "strategies": [
        {
            "id": "turtle_entry",
            "logic": "all",
            "conditions": [
                {
                    "condition_id": "VolatilityBreakout",
                    "params": {"k": 0.5}
                },
                {
                    "condition_id": "ATR",
                    "params": {"period": 20, "threshold": 0.5}
                }
            ]
        },
        {
            "id": "turtle_exit",
            "logic": "any",
            "conditions": [
                {
                    "condition_id": "RSI",
                    "params": {"period": 14, "overbought": 70}
                }
            ]
        }
    ],
    "orders": {
        "new_orders": [
            {
                "strategy_id": "turtle_entry",
                "conditions": [
                    {
                        "condition_id": "FixedFractional",
                        "params": {
                            "risk_per_trade": 0.02,
                            "max_position": 0.1
                        }
                    }
                ]
            }
        ]
    }
}

# 실행
async def main():
    pg = ProgramGarden()
    pg.set_listener(listener)
    await pg.run_async(system=system)

if __name__ == "__main__":
    asyncio.run(main())
```

### 29.5 에러 핸들링 최적화

```python
from programgarden_core.exceptions import (
    OrderException,
    ConditionExecutionException,
)

async def execute_with_retry(func, max_retries=3):
    for attempt in range(max_retries):
        try:
            return await func()
        except OrderException as e:
            if attempt == max_retries - 1:
                raise
            await asyncio.sleep(2 ** attempt)  # 지수 백오프
        except ConditionExecutionException:
            # 조건 실행 오류는 재시도하지 않음
            raise
```

## 31. 디버깅 및 트러블슈팅
================================================================================

### 31.1 일반적인 오류와 해결 방법

#### AppKeyException: APPKEY_NOT_FOUND
```
원인: appkey 또는 appsecretkey가 설정되지 않음
해결:
1. .env 파일에 APPKEY, APPSECRET 확인
2. load_dotenv() 호출 여부 확인
3. 환경 변수명 오타 확인
```

#### LoginException: LOGIN_ERROR
```
원인: 로그인 실패
해결:
1. appkey/appsecretkey 유효성 확인
2. LS증권 API 키 만료 여부 확인
3. 네트워크 연결 상태 확인
4. 증권사 서버 점검 시간 확인 (보통 04:00~06:00)
```

#### TokenException: TOKEN_ERROR
```
원인: OAuth 토큰 발급 실패
해결:
1. appkey/appsecretkey 재확인
2. API 키 권한 확인 (해외주식/해외선물)
3. 토큰 캐시 삭제 후 재시도
```

#### OrderException: ORDER_ERROR
```
원인: 주문 처리 실패
해결:
1. 계좌번호 확인
2. 주문 가능 수량/금액 확인
3. 시장 개장 시간 확인
4. 종목코드 유효성 확인
```

#### ConditionExecutionException: CONDITION_EXECUTION_ERROR
```
원인: 전략 조건 실행 중 오류
해결:
1. 조건 ID 존재 여부 확인

3. 파라미터 타입 확인 (int vs float)

```

### 35.1 사용 가능한 이벤트 목록

| 이벤트 | 트리거 시점 | 콜백 인자 |
|--------|-------------|-----------|
| on_system_ready | 시스템 초기화 완료 | symbols |
| on_strategy_start | 전략 평가 시작 | strategy_id |
| on_strategy_executed | 전략 평가 완료 | strategy_id, results |
| on_condition_evaluated | 개별 조건 평가 완료 | condition_id, result |
| on_order_created | 주문 객체 생성 | order_info |
| on_order_submitted | 주문 제출 | order_info |
| on_order_filled | 체결 완료 | fill_info |
| on_order_cancelled | 주문 취소 | cancel_info |
| on_order_rejected | 주문 거부 | reject_info |
| on_error | 오류 발생 | error |
| on_system_shutdown | 시스템 종료 | reason |

### 35.2 이벤트 리스너 전체 예제

```python
from programgarden import ProgramGarden, PGListener

listener = PGListener()

@listener.on_system_ready
def on_ready(symbols):
    print(f"=== 시스템 준비 완료 ===")
    print(f"대상 종목: {len(symbols)}개")
    for s in symbols:
        print(f"  - {s['symbol']} ({s['exchcd']})")

@listener.on_strategy_start
def on_strategy_start(strategy_id):
    print(f"\n[전략 시작] {strategy_id}")

@listener.on_condition_evaluated
def on_condition(condition_id, result):
    status = "✓" if result["success"] else "✗"
    print(f"  {status} 조건: {condition_id}")
    if result.get("data"):
        print(f"    데이터: {result['data']}")

@listener.on_strategy_executed
def on_strategy_done(strategy_id, results):
    success_count = sum(1 for r in results if r["success"])
    print(f"[전략 완료] {strategy_id}: {success_count}/{len(results)} 통과")

@listener.on_order_created
def on_order_created(order_info):
    print(f"\n[주문 생성]")
    print(f"  종목: {order_info['symbol']}")
    print(f"  방향: {order_info['side']}")
    print(f"  수량: {order_info['qty']}")
    print(f"  가격: {order_info['price']}")

@listener.on_order_submitted
def on_submitted(order_info):
    print(f"[주문 제출] 주문번호: {order_info['ord_no']}")

@listener.on_order_filled
def on_filled(fill_info):
    print(f"[체결 완료]")
    print(f"  종목: {fill_info['symbol']}")
    print(f"  체결가: {fill_info['price']}")
    print(f"  체결수량: {fill_info['qty']}")
    print(f"  체결시각: {fill_info['time']}")

@listener.on_order_cancelled
def on_cancelled(cancel_info):
    print(f"[주문 취소] {cancel_info['symbol']} 주문번호: {cancel_info['ord_no']}")

@listener.on_order_rejected
def on_rejected(reject_info):
    print(f"[주문 거부]")
    print(f"  종목: {reject_info['symbol']}")
    print(f"  사유: {reject_info['reason']}")

@listener.on_error
def on_error(error):
    print(f"\n[에러 발생]")
    print(f"  코드: {error.code if hasattr(error, 'code') else 'N/A'}")
    print(f"  메시지: {str(error)}")

@listener.on_system_shutdown
def on_shutdown(reason):
    print(f"\n=== 시스템 종료 ===")
    print(f"종료 사유: {reason}")

# ProgramGarden에 리스너 등록
pg = ProgramGarden()
pg.set_listener(listener)
```

```python
async def subscribe_gsc():
    client = ls.overseas_stock().real()
    await client.connect()
    
    gsc = client.GSC()
    gsc.add_gsc_symbols(symbols=["82TSLA", "82AAPL"])
    
    def on_gsc(data):
        """
        data 구조:
        {
            "symbol": "TSLA",           # 종목코드
            "price": 248.50,            # 체결가
            "volume": 100,              # 체결량
            "time": "093045123",        # 체결시각 (HHMMSSmmm)
            "change": 2.50,             # 전일대비
            "change_rate": 1.02,        # 등락률 (%)
            "cum_volume": 1234567,      # 누적거래량
            "open": 246.00,             # 시가
            "high": 249.00,             # 고가
            "low": 245.50,              # 저가
            "prev_close": 246.00        # 전일종가
        }
        """
        print(f"{data['symbol']}: {data['price']} ({data['change_rate']}%)")
    
    gsc.on_gsc_message(on_gsc)
    await asyncio.sleep(3600)  # 1시간 구독
```

```python
async def managed_websocket():
    client = ls.overseas_stock().real()
    
    # 연결
    await client.connect()
    print("WebSocket 연결됨")
    
    try:
        gsc = client.GSC()
        gsc.add_gsc_symbols(symbols=["82TSLA"])
        gsc.on_gsc_message(lambda d: print(d))
        
        # 연결 유지
        while True:
            await asyncio.sleep(30)
            # heartbeat 또는 상태 체크
            
    except asyncio.CancelledError:
        print("취소됨")
    finally:
        # 구독 해지
        gsc.on_remove_gsc_message()
        # 연결 종료
        await client.close()
        print("WebSocket 연결 종료")
```

## 37. 용어 사전
================================================================================

### 37.1 일반 용어

| 용어 | 설명 |
|------|------|
| DSL | Domain Specific Language, 특정 도메인을 위한 언어 |
| 전략 (Strategy) | 매매 조건과 로직을 정의한 규칙 집합 |
| 조건 (Condition) | 개별 매매 판단 기준 |
| 파라미터 (Parameter) | 조건에 전달하는 설정값 |
| 리스너 (Listener) | 이벤트 발생 시 실행되는 콜백 함수 |

### 37.2 주문 관련 용어

| 용어 | 설명 |
|------|------|
| 신규주문 (New Order) | 새로운 매수/매도 주문 |
| 정정주문 (Modify Order) | 기존 주문의 가격/수량 변경 |
| 취소주문 (Cancel Order) | 미체결 주문 취소 |
| 지정가 (Limit Order) | 특정 가격으로 주문 |
| 시장가 (Market Order) | 현재 시장가로 즉시 체결 |
| 체결 (Fill/Execution) | 주문이 실제로 거래됨 |
| 미체결 (Unfilled) | 아직 체결되지 않은 주문 |

### 37.3 시장 관련 용어

| 용어 | 설명 |
|------|------|
| 호가 (Quote) | 매수/매도 희망 가격 |
| 스프레드 (Spread) | 매수-매도 호가 차이 |
| 거래량 (Volume) | 거래된 수량 |
| 시가 (Open) | 당일 첫 거래 가격 |
| 고가 (High) | 당일 최고 가격 |
| 저가 (Low) | 당일 최저 가격 |
| 종가 (Close) | 당일 마지막 거래 가격 |
| 전일종가 (Previous Close) | 전날 종가 |

### 37.4 기술적 분석 용어

| 용어 | 설명 |
|------|------|
| SMA | 단순이동평균 (Simple Moving Average) |
| EMA | 지수이동평균 (Exponential Moving Average) |
| RSI | 상대강도지수 (Relative Strength Index) |
| MACD | 이동평균수렴확산 (Moving Average Convergence Divergence) |
| ATR | 평균진폭 (Average True Range) |
| 골든크로스 | 단기 이평선이 장기 이평선 상향 돌파 |
| 데드크로스 | 단기 이평선이 장기 이평선 하향 돌파 |
| 과매수 (Overbought) | 지표상 가격이 과도하게 상승한 상태 |
| 과매도 (Oversold) | 지표상 가격이 과도하게 하락한 상태 |

### 37.5 선물 관련 용어

| 용어 | 설명 |
|------|------|
| 만기 (Expiration) | 선물 계약 종료일 |
| 롤오버 (Rollover) | 만기 전 다음 월물로 이전 |
| 롱 (Long) | 매수 포지션 |
| 숏 (Short) | 매도 포지션 |
| 증거금 (Margin) | 선물 거래를 위한 담보금 |
| 미결제약정 (Open Interest) | 청산되지 않은 계약 수 |
| 틱 (Tick) | 최소 가격 변동 단위 |

### 37.6 증권사 API 용어

| 용어 | 설명 |
|------|------|
| TR 코드 | Transaction 코드, API 요청 식별자 |
| appkey | API 인증 키 |
| appsecretkey | API 시크릿 키 |
| OAuth 토큰 | 인증된 액세스 토큰 |
| WebSocket | 실시간 양방향 통신 프로토콜 |
| 동기 요청 | 응답을 기다리는 요청 (req) |
| 비동기 요청 | 응답을 기다리지 않는 요청 (req_async) |

```python
import re

def validate_cron_expression(expr: str) -> bool:
    """
    6필드 크론 표현식 검증
    초 분 시 일 월 요일
    """
    cron_pattern = r'^(\*|[0-9,\-\/]+)\s+(\*|[0-9,\-\/]+)\s+(\*|[0-9,\-\/]+)\s+(\*|[0-9,\-\/\?]+)\s+(\*|[0-9,\-\/]+)\s+(\*|[0-9a-zA-Z,\-\/]+)$'
    return bool(re.match(cron_pattern, expr))

# 유효한 예시
valid_expressions = [
    "* * * * * *",           # 매초
    "0 * * * * *",           # 매분
    "0 0 * * * *",           # 매시간
    "0 30 23 * * mon-fri",   # 평일 23:30
    "0 */5 * * * *",         # 5분마다
]
```

```python
async def stream_trades(symbols: list):
    ls = LS()
    ls.login(os.getenv("APPKEY"), os.getenv("APPSECRET"))
    
    client = ls.overseas_stock().real()
    await client.connect()
    
    gsc = client.GSC()
    gsc.add_gsc_symbols(symbols=symbols)
    
    def on_trade(data):
        print(f"[{data['time']}] {data['symbol']}: "
              f"${data['price']} ({data['change_rate']:+.2f}%)")
    
    gsc.on_gsc_message(on_trade)
    
    try:
        await asyncio.sleep(3600)  # 1시간
    finally:
        gsc.on_remove_gsc_message()
        await client.close()

asyncio.run(stream_trades(["82TSLA", "82AAPL"]))
```

```python
import asyncio
from datetime import datetime

class HealthChecker:
    def __init__(self):
        self.last_heartbeat = datetime.now()
        self.is_healthy = True
    
    async def heartbeat(self):
        """주기적으로 호출되어야 함"""
        self.last_heartbeat = datetime.now()
    
    async def check(self) -> dict:
        """헬스 상태 반환"""
        elapsed = (datetime.now() - self.last_heartbeat).seconds
        self.is_healthy = elapsed < 60  # 60초 이내
        
        return {
            "status": "healthy" if self.is_healthy else "unhealthy",
            "last_heartbeat": self.last_heartbeat.isoformat(),
            "elapsed_seconds": elapsed
        }

# HTTP 헬스 엔드포인트 (FastAPI 예시)
from fastapi import FastAPI

app = FastAPI()
```

### 42.1 백테스팅 개요

ProgramGarden은 실시간 거래에 초점을 맞추지만, 전략 검증을 위해
외부 백테스팅 라이브러리와 연동할 수 있습니다.

### 42.2 Backtrader 연동 예시

```python
import backtrader as bt
import pandas as pd

class SMAStrategy(bt.Strategy):
    params = (
        ('short_period', 5),
        ('long_period', 20),
    )
    
    def __init__(self):
        self.sma_short = bt.indicators.SMA(
            self.data.close, period=self.params.short_period
        )
        self.sma_long = bt.indicators.SMA(
            self.data.close, period=self.params.long_period
        )
        self.crossover = bt.indicators.CrossOver(
            self.sma_short, self.sma_long
        )
    
    def next(self):
        if not self.position:
            if self.crossover > 0:  # 골든크로스
                self.buy()
        elif self.crossover < 0:    # 데드크로스
            self.sell()

# 백테스트 실행
cerebro = bt.Cerebro()
cerebro.addstrategy(SMAStrategy, short_period=5, long_period=20)

# 데이터 로드
data = bt.feeds.PandasData(dataname=pd.read_csv('tsla.csv'))
cerebro.adddata(data)

cerebro.broker.setcash(100000)
cerebro.broker.setcommission(commission=0.001)

results = cerebro.run()
cerebro.plot()
```

```python
def get_daily_summary(date: str = None):
    if date is None:
        date = datetime.now().strftime("%Y-%m-%d")
    
    c = conn.cursor()
    c.execute('''
        SELECT 
            COUNT(*) as total_trades,
            SUM(CASE WHEN side = 'buy' THEN quantity * price ELSE 0 END) as buy_amount,
            SUM(CASE WHEN side = 'sell' THEN quantity * price ELSE 0 END) as sell_amount
        FROM trades
        WHERE DATE(timestamp) = ?
    ''', (date,))
    
    return c.fetchone()

def get_symbol_performance():
    c = conn.cursor()
    c.execute('''
        SELECT 
            symbol,
            COUNT(*) as trade_count,
            SUM(CASE WHEN side = 'sell' THEN quantity * price 
                     WHEN side = 'buy' THEN -quantity * price 
                END) as realized_pnl
        FROM trades
        GROUP BY symbol
        ORDER BY realized_pnl DESC
    ''')
    return c.fetchall()
```

```python
import pandas as pd

def load_trades_to_df():
    return pd.read_sql_query(
        "SELECT * FROM trades ORDER BY timestamp",
        conn,
        parse_dates=['timestamp']
    )

def analyze_trades(df):
    # 일별 거래 통계
    daily = df.groupby(df['timestamp'].dt.date).agg({
        'id': 'count',
        'price': ['mean', 'sum'],
        'quantity': 'sum'
    })
    
    # 종목별 거래 통계
    by_symbol = df.groupby('symbol').agg({
        'id': 'count',
        'quantity': 'sum'
    }).sort_values('id', ascending=False)
    
    # 시간대별 거래 분포
    hourly = df.groupby(df['timestamp'].dt.hour)['id'].count()
    
    return daily, by_symbol, hourly
```

```python
class DrawdownManager:
    def __init__(self, max_drawdown: float = 0.1):
        self.max_drawdown = max_drawdown
        self.peak_equity = 0
        self.current_equity = 0
    
    def update(self, equity: float) -> bool:
        self.current_equity = equity
        self.peak_equity = max(self.peak_equity, equity)
        
        drawdown = (self.peak_equity - self.current_equity) / self.peak_equity
        
        if drawdown >= self.max_drawdown:
            return False  # 거래 중지
        return True  # 거래 계속

# 사용 예시
```

### 49.4 변동성 기반 포지션 사이징

```python
def calculate_position_size(
    account_equity: float,
    risk_per_trade: float,
    entry_price: float,
    stop_loss_price: float,
    atr: float = None
) -> int:
    """ATR 기반 포지션 사이징"""
    
    if atr:
        # ATR 기반 손절가
        stop_loss_price = entry_price - (2 * atr)
    
    risk_amount = account_equity * risk_per_trade
    risk_per_share = abs(entry_price - stop_loss_price)
    
    if risk_per_share == 0:
        return 0
    
    position_size = int(risk_amount / risk_per_share)
    return position_size

# 사용 예시
position = calculate_position_size(
    account_equity=100000,
    risk_per_trade=0.02,  # 2%
    entry_price=250.0,
    stop_loss_price=237.5  # -5%
)
print(f"적정 포지션: {position}주")
```

### Q8: API 키는 어디서 발급받나요?

```
A: LS증권 OpenAPI 포털에서 발급받습니다:
1. https://openapi.ls-sec.co.kr/ 접속
2. 회원가입 및 로그인
3. API 신청 메뉴에서 appkey, appsecretkey 발급
4. 해외주식/해외선물 권한 별도 신청 필요
```

### Q9: 토큰이 계속 만료됩니다.
```
A: ProgramGarden Finance는 토큰 자동 갱신을 지원합니다.
수동 관리가 필요하면:

from programgarden_finance import LS

ls = LS()
ls.login(appkey=..., appsecretkey=...)

# 토큰 상태 확인
if ls.is_token_expired():
    ls.refresh_token()
```
